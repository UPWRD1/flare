# Intentionality over Simplicity
This is the key distinction! Code should be intentional, not just simple. Therefore, the language must make it HARD to write code that is unintentional!


# Minimize complexity
Complex-looking code should be a sign of complex behavior. Simple ideas should look simple. This naturally results in Readability, and Aesthetic

# Self-enforcing.
The language must try to prevent antipatterns. "The only way is the right way"
